**Phaser 이관 = 보류 (에이전트 제안과 다른 결정)**: M2도 DOM+Canvas 유지. 근거 — ① 엔진이 순수 TS로 분리돼 이관 비용이 언제든 통제됨 ② 잉크 라인 판타지(정지 일러스트+이펙트)는 현 스택으로 충분 ③ M2에 프레임워크 이관을 얹으면 데모 일정 리스크. 단, M2 구현 중 명확한 한계(성능/연출 복잡도)에 부딪히면 **멈추고 보고** — 그때 재평가. NFR-011은 "웹 표준 스택(TS+Vite), 렌더링 프레임워크는 필요 시 도입"으로 개정.

## 2. M2 — 데모 (목표 3주): 프롤로그~스테이지 1 완결 30~40분

### T2-01. 노드 맵 + 스테이지 구조
- 스테이지 1을 3구역(전반/중반/보스 관문)으로, 노드 타입 5종(전투/수련/이벤트/상점/휴식) 분기 맵. stages/*.json 데이터 정의. 결전은 15분 세션당 1회 이하 배치 규칙(NFR-008)
- 완료 기준: 노드 맵 UI에서 노드 선택→해당 씬 진입→복귀 순환

### T2-02. 저장 시스템
- localStorage: 구역 진입 체크포인트(패배 복귀) + 노드 완료 이어하기 자동저장(별도 슬롯), version 필드, 설계서 §10 스키마
- 완료 기준: 앱 재시작 후 이어하기 정상, 패배 시 구역 시작 복귀, 손실 최대 1노드

### T2-03. 조우전 + 정예전
- 조우전: 실시간, 적 방향 스와이프(검결 1타), 등급 없음. 섬광 일격(우위 임계 balance). 정예전: 간소 합 3~5회, 요구치 완화
- 완료 기준: 세 전투 무게가 노드 타입별로 구분 동작

### T2-04. 경제/아이템/레벨
- 골드(전투 보상·의뢰), 상점 노드(아이템 3종+검 3종, 설계서 §9.2 초안가), 아이템은 관찰 페이즈만 사용, 레벨/경험치(HP+10/위력+2), items.json 획득 경로 필드
- 완료 기준: 벌기→사기→쓰기 루프 완주, 레벨업 반영

### T2-05. 스토리/대화 + 유파 선택 이벤트
- 대화 시스템(초상화+텍스트, 스킵/자동), 프롤로그+스테이지1 대본 초안을 트리트먼트(몰락 검술 명가 복수극, 인물 5명 이내) 기반으로 작성 — **대본 초안은 자동 진행하지 말고 출력해서 검토받을 것** (스토리는 설계 검토 대상)
- 유파 선택을 스토리 이벤트로("너는 어느 손에 검을 쥘 것이냐"), 프롤로그 첫 획 관대 판정
- 완료 기준: 프롤로그→입문→스테이지1 진입이 대화 흐름으로 이어짐

### T2-06. 적 확장 + 보스
- 적 4~5종(야수/고블린류 중심, enemies.json), 스테이지1 보스 1종 = 특정 획의 달인 기믹(FR-CBT-011)
- 완료 기준: 보스전에서 해당 획 숙련 여부가 난이도로 체감

### T2-07. 타이틀/설정
- 타이틀(시작/이어하기/설정/피드백 버튼 — URL은 사용자 제공), 설정(사운드/햅틱/판정 난이도 관대·표준/대화 속도/가상키 크기), 첫 진입 "탭하여 시작"(AudioContext 게이트 겸용)
- 완료 기준: 타이틀→새 게임/이어하기 분기 동작

### T2-08. 밸런스 절대값 + 튜닝
- 적 HP/공격 절대값을 데미지 체인과 맞춰 산정(결전 3~5분, 정예 1~2분 목표 역산). 테스터 피드백 반영: simulMs/beatMs/lenBand/OI-16 임계
- 완료 기준: 스테이지1 완주 시간 30~40분 실측

### T2-09. M2 검증 + 리포트
- 완주 실기(폰/PC), 저장/이어하기, 테스터 결과 취합, M3 리스크. 한 덩어리 출력

### T2-10 (사용자 승인 대기). 최소 플레이 로그
- FR-OPS-003 전진 배치: 스테이지 이탈 지점 + 획별 평균 정확도만 Supabase 익명 수집. 승인 시 T2-07 이후 진행
