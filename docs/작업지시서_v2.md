## 0. M0 마감 판정

**조건부 통과.** 기능 DoD 전 항목 충족 (획 3종 양방식 판정, 등급/사운드/오버레이, 연풍참, 좌수 미러, 채널 중재). 잔여 1건 = **테스터 3인 "다시 긋고 싶다"** — 실사용자 검증이라 개발과 병행 가능하므로 M1 착수를 막지 않는다. 단 **M1 완료 전까지 반드시 실시** (미실시 상태로 M2 진입 금지 — "납득 가능한 실패" 체감은 리스크 1순위 항목).

리포트의 설계 변경 6건은 전부 **승인**. OI-12는 M1로 이관, OI-16은 계속 오픈 (아래 §2 참조).

## 2. M0에서 넘어온 설계 결정 (M1 작업의 전제)

| 부채/이슈 | 결정 |
|---|---|
| 리듬 self-referenced (외부 클럭 미동기) | **유지.** 결전 응수 페이즈는 시간 정지 상태의 입력이므로 첫 입력=다운비트 방식이 설계 §6.2와 정합. 수련 메트로놈만 외부 클럭에 입력을 맞추는 연습 UI로 구현 (판정은 동일 함수) |
| 찌르기 vs minLenRatio 0.12 충돌 | 템플릿별 `minLenRatio` 오버라이드 필드 허용 (찌르기 초안 0.05). 거부비 0.035는 전역 유지 |
| 좌수 미러 시 새 획(diag_dl) | 별도 문제 아님 — 획 8종 완성 시 h_lr↔h_rl, diag_dr↔diag_dl이 상호 미러 매핑이 됨. mirrorX 변환을 "템플릿 생성"이 아닌 "획ID 매핑 테이블"로 단순화 가능 |
| 콤보창 1500 vs 확정대기 640 잠식 | 검결의 검술 성립은 획 단위 타임스탬프(확정 시점이 아닌 첫 입력 시점) 기준으로 계산해 잠식 제거. 튜닝 잔여 시 balance |
| powerBonus 표기만 | T1-05에서 실데미지 곱셈 연결 |
| beatMs 400 임의값 | 검술별 rhythm 타임라인을 techniques 데이터로 이동 (T1-03). 400은 기본값 |
| OI-16 보너스 임계 ≤100ms 관대 | 당분간 유지. 실데미지 연결 + 테스터 피드백 후 퍼펙트(≤50ms)로 좁힐지 결정 |
| AudioContext 게이팅 | 메트로놀/소리 토글이 사용자 제스처이므로 토글 시 resume 호출로 충분. 첫 진입 시 "탭하여 시작" 게이트 화면을 M1 타이틀에 겸용 |

## 3. M1 — 코어 (목표 3주)

**목표 (SRS §7)**: 획 8종 + 검술 6종 + 유파 2종 + 수련 씬 + 결전 페이즈 전투 + 마나 공식 + 승급 시험 1종.

### T1-01. 프로젝트 전환 + 엔진 분리
- Phaser 3 + TypeScript + Vite 셋업. M0 순수부(judgeStroke/judgeRhythm/ComboTracker/미러)를 `src/engine/` 모듈로 이식, DOM 계층과 분리
- BALANCE/STROKE_TEMPLATES/TECHNIQUES/STYLES를 `src/data/*.json`으로 외부화
- **완료 기준**: M0의 node 회귀 29종이 새 구조에서 전부 통과 (회귀 테스트를 vitest로 이관). GitHub Pages 배포 파이프라인이 Vite 빌드 기준으로 동작
- **선행**: T1-00

### T1-02. 획 8종 완성 (검로)
- 추가 5종: 횡베기(우→좌), 사선(↙), 올려베기, 찌르기(minLenRatio 오버라이드 0.05), 원무(원형 템플릿 + 원형도 지표 — 직선도 30%를 획 타입별로 직선도/원형도로 분기)
- **완료 기준**: 8종 상호 오분류율이 합성 테스트에서 수용 가능(특히 찌르기↔횡베기, 원무↔사선), 실기에서 각 획 그레이트+ 달성 가능
- **선행**: T1-01

### T1-03. 검결 8종 + 검술별 리듬
- 커맨드 8종: →← / ↓+← / ↓↑ / →→ / 원무 →↓←↑. 검술별 rhythm 타임라인을 techniques.json으로 이동
- **OI-12 검증**: 원무 →↓←↑가 다른 획 대비 과하게 어려운지 실기 판정 → 결과를 WORKLOG + SRS OI-12에 기록
- **완료 기준**: 8종 전부 키보드/가상패드에서 성립, 원무 판정 소감 기록
- **선행**: T1-01

### T1-04. 유파 정식화
- styles.json + StyleManager: 획ID 미러 매핑, 검술별 획 조합 오버라이드(연풍참 좌수판), 가상패드 ↑↓ 클러스터 전환
- **완료 기준**: 유파 전환 시 8획 전체와 검술 6종이 좌수 조합으로 동작
- **선행**: T1-02, T1-03

### T1-05. 검술 6종 + 전투 수치 연결
- 일자베기/십자베기/연풍참/낙뢰참/회천무/섬광자돌 (설계서 §4.1 초안 조합). 미스 페널티(마나 30% 소실 + 0.8s 경직), 퍼펙트 아츠 +20%, **검결 +10% 실데미지 곱셈 반영**
- **완료 기준**: 6종 전부 성립하고 데미지 수치가 공식대로 산출 (로그로 검증)
- **선행**: T1-02, T1-03

### T1-06. 수련 씬
- 스승 시범 궤적 가이드라인 → 반복 시 페이드아웃 → 소거(암기 확인). 굿+ 3연속 통과. 스승 대사 등급 연동(퍼펙트/미스 전용). 검결 수련은 메트로놈 + 비트 마커(끄기 가능)
- **완료 기준**: 획 1종의 수련 흐름이 처음부터 통과까지 완주 가능
- **선행**: T1-01

### T1-07. 결전 전투 (관찰→응수→해소)
- 페이즈 FSM + 적 1종(enemies.json, 공격 예고/방향). 응수: 시간 정지 + 입력(검로/검결 병행, self-referenced 리듬). 패링(직각/반대 획 → 반격 합). 기본 공격 무마나, 마나 회복=합 진행+퍼펙트. HP/승패
- InputArbiter v2를 여기서 정식화: drop 모델 → 우선순위/큐/캔슬 (빠른 응수에서 의도 입력 드롭 제거)
- **완료 기준**: 결전 1판을 처음부터 승/패까지 플레이 가능, 소리/햅틱만으로 응수 진행 가능
- **선행**: T1-05

### T1-08. 승급 시험 1종 (마나 각성)
- 지정 4획 연속 시연, 획 간 간격 ≤2.0s, 평균 70+, 미스 즉시 중단. 불합격 시 최저 획 짚는 대사 + 재도전
- **완료 기준**: 합격/불합격/재도전 흐름 완주
- **선행**: T1-02

### T1-09. M1 검증 + 리포트
- vitest 회귀 전체 + 실기(폰/PC). **테스터 3인 플레이테스트 결과 취합** (M0 잔여 DoD — 사용자가 섭외). OI-12/16 결론, 오분류율, M2 리스크를 WORKLOG에 리포트로 정리해 한 덩어리 출력
- **선행**: 전체

**의존 순서**: T1-00 → T1-01 → (T1-02, T1-03, T1-06 병렬) → T1-04 → T1-05 → (T1-07, T1-08 병렬) → T1-09
