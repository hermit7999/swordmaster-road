## 1. 범위
모바일+PC 웹 제스처 검술 액션 RPG. M0 = 획 3종 · 검로/검결 판정 · 등급/사운드/오버레이 · 연풍참 · 좌수 미러 · 채널 중재. (본편/전투/경제는 M1~.)

## 2. 입력 요구사항 (FR-INP)

- **FR-INP-002** 검로 궤적 수집: Pointer Events로 터치=마우스 동일 처리. 좌표는 `clientX − 캔버스 rect`(offsetX 대체, 변환/DPR 안전). 실시간 잉크 궤적 렌더.
- **FR-INP-004** 방향 정확도: 궤적 방향 정밀도를 **리샘플 각 구간의 국소 방향과 템플릿 방향의 일치도 평균**으로 산출한다(가중 40%). *정정: v1.1의 "시작→끝 순(net) 벡터" 산식은 흔들린 획을 배드로 판정하지 못해 폐기.*
- **FR-INP-006** 검결 커맨드 캡처: 방향 입력을 버퍼링해 (a) 순차(예 ←후→) (b) 동시키(↓+→ → 대각 접기, `simulMs` 70ms) 를 인식. 키보드(방향키/WASD, `e.repeat` 무시) + 가상패드(멀티터치). 산출 = (획ID 후보, 타임스탬프 시퀀스).
- **FR-INP-007** 커맨드 확정: 완전 일치 시 즉시 확정, 아니면 `commandResolveMs`(640ms) 뒤 확정.
- **FR-INP-010** 채널 중재: 검로/검결 동시 활성, 행동을 먼저 시작한 채널이 점유(occupancy), 종료 시 해제. 겹치는 후발 입력은 무시. 백스톱 `arbiterStaleMs` 2000ms.
- **FR-INP-012** 가상패드 레이아웃: 좌 클러스터 ↖←↙ / 우 클러스터 ↗→↘, ↑↓는 유파 클러스터(우수=우측/좌수=좌측). 중앙=제스처 캔버스.
- **FR-INP-013** 검결 리듬 판정: 이상 비트 타임라인 대비 타이밍 오차(퍼펙트±50/그레이트±100/굿±180/배드±280ms, 순서·개수 오류=미스). self-referenced(첫 입력=다운비트).
- **FR-INP-014** 검결 위력 보상: 다입력 검결을 정밀하게 성공하면 위력 +10%. **적용 조건: raw 입력 ≥ 2 && 등급 ≠ 미스 && 최대 타이밍오차 ≤ 100ms(그레이트 창). 패드 단일 대각 press는 제외.** (M0: HUD 표기만. 실데미지 곱셈은 M1 T1-05.)

## 3. 판정 요구사항 (FR-JDG)

- **FR-JDG-001** 순수 함수 `judgeStroke` — DOM 비의존, M1 이식 대상.
- **FR-JDG-002** 크기 정규화, 회전 비정규화(방향=획 정체성).
- **FR-JDG-003** 탭/짧은 획 거부(rejectRatio 0.035 / minLenRatio 0.12, 획별 오버라이드 허용).
- **FR-JDG-004** 정확도 = 방향 40%(국소 일치도) + 직선 30%((현/경로)²) + 속도 15% + 완주 15%.
- **FR-JDG-005** 순수 함수 `judgeRhythm` — 검결 타이밍 판정.

## 4. 피드백 요구사항
- **FR-FBK-001** 등급별 검명 5종(음색 구분) + 모바일 햅틱 — 화면 안 봐도 등급 인지.
- **FR-FBK-002** 실패 시 내 궤적 vs 이상 궤적 비교 오버레이("납득 가능한 실패").
- **FR-FBK-003** 메트로놈(목검 비트): Web Audio 틱 + 비트 펄스 + 토글. AudioContext는 토글(사용자 제스처) 시 resume.

## 5. 이벤트 규격
판정 이벤트: `{strokeId, accuracy, grade, inputMode('gesture'|'command'), timestamp}` (+선택 `breakdown`, `powerBonus`). 하위 로직은 입력 방식·유파 무관.
