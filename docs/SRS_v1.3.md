## 1. 범위
모바일+PC 웹 제스처 검술 액션 RPG. 가로 고정. 단일 코드베이스(TS+Vite, DOM+Canvas).
- M0: 획 3종·검로/검결 판정·등급/사운드/오버레이·연풍참·좌수 미러·채널 중재.
- M1: 획 8종·검술 6종·유파 2종·수련 씬·결전 전투·마나 공식·승급 시험 1종.
- M2(데모): 노드맵·저장·조우/정예/결전·경제/아이템/레벨·스토리·적 확장/보스·타이틀/설정. 프롤로그~스테이지1 완결(30~40분).

## 2. 입력 요구사항 (FR-INP)
- **FR-INP-002** 검로 궤적 수집: Pointer Events(터치=마우스), 좌표 `clientX−rect`(변환/DPR 안전), 실시간 잉크 렌더.
- **FR-INP-004** 방향 정확도: 리샘플 각 구간 국소 방향과 템플릿 방향 일치도 평균(가중 40%). *v1.1 순 벡터 산식 폐기.*
- **FR-INP-006** 검결 커맨드 캡처: 순차/동시키(↓+→ 대각 접기, `simulMs` 70ms) 인식. 키보드(방향키/WASD, `e.repeat` 무시)+가상패드(멀티터치). 산출=(획ID 후보, 타임스탬프).
- **FR-INP-007** 커맨드 확정: 완전 일치 즉시, 아니면 `commandResolveMs`(640ms) 뒤.
- **FR-INP-010** 채널 중재(InputArbiter v2): 검로/검결 동시 활성, 먼저 시작한 채널 점유. **아직 안 그린(점만 찍힌) 제스처는 커맨드가 선점(취소/우선순위)** — 빠른 응수 드롭 방지. 백스톱 `arbiterStaleMs` 2000ms.
- **FR-INP-012** 가상패드: 좌 ↖←↙ / 우 ↗→↘, ↑↓=유파 클러스터(우수=우측/좌수=좌측).
- **FR-INP-013** 검결 리듬 판정: 이상 비트 대비 타이밍(퍼펙트±50/그레이트±100/굿±180/배드±280ms). **동시 그룹(동일 오프셋)은 순서 무관 멀티셋 매칭**(↓+→에서 →가 먼저 들어와도 정상). self-referenced.
- **FR-INP-014** 검결 위력 보상 +10%: raw≥2 && 등급≠미스 && 최대오차≤100ms. 패드 단일 대각 제외. **검술 단위 적용은 all-or-nothing**(FR-TEC-002).

## 3. 판정 요구사항 (FR-JDG)
- **FR-JDG-001** 순수 함수 `judgeStroke`(DOM 비의존, src/engine).
- **FR-JDG-002** 크기 정규화, 회전 비정규화(방향=획 정체성).
- **FR-JDG-003** 탭/짧은 획 거부(rejectRatio 0.035 / minLenRatio 획별 오버라이드, 사전거부 바닥=최소 minLenRatio).
- **FR-JDG-004** 정확도 = 방향 40%(국소 일치도) + **형태 30%** + 속도 15% + 완주 15%.
- **FR-JDG-005** 순수 함수 `judgeRhythm` — 검결 타이밍.
- **FR-JDG-006 (M1)** 형태 분기: `shapeType`(line/circle). line=직선도 (현/경로)². **circle=원형도**(반지름 일관성 50%+각도 스윕 30%+폐합 20%), 방향=회전방향 일치+단조성(반대방향=0). 원 게이트=스윕≥200°+반지름CV≤0.35+폐합≤0.6.
- **FR-JDG-007 (M1)** lenBand 판별: 형태 동률 후보(tieEps 0.02)를 캔버스 대각 대비 길이비 밴드로 구분(찌르기[0.035,0.16] vs 횡베기[0.16,1.0]).

## 4. 전투/검술 요구사항 (FR-TEC / FR-CBT) — M1
- **FR-TEC-001** 검술 6종(techniques.json): 다중 검술 동시 추적(접두어 공유, 긴 조합 우선), 기본 획 노이즈 prune.
- **FR-TEC-002** 데미지 = base × (평균≥95?1.2:1.0) × 검결보너스. 검결 +10%는 **아츠 단위 all-or-nothing**(전 획 검결+보너스 시 ×1.1). `commandBonusMode` 플래그로 proportional(충족획/전체) 전환 가능. 마나 = cost×(2−평균/100).
- **FR-TEC-003** 미스 페널티: 진행 중 미스 = 발동 실패, 마나 30%(cost×0.3) 소실 + 경직 800ms. **페널티 대상 검술 = 진행 중 접두어 후보 중 최장**(공유 접두어 모호성 규칙).
- **FR-CBT-001** 결전 페이즈 FSM: 관찰(觀察, 예고+방향 오디오 큐)→응수(應手, 시간 정지, 첫 획=패링)→해소(解消, 반격/피격). HP/승패.
- **FR-CBT-002** 패링: 적 공격의 반대/직각 획(데이터 지정 counter)을 굿以上으로 성공 시 반격. 기본 패링 무마나, 성공 시 마나 회복(퍼펙트 더).
- **FR-CBT-011 (M2)** 보스 = 특정 획의 달인 기믹(해당 획 숙련이 난이도로 체감).

## 5. 씬/진행 요구사항 (M1/M2)
- **FR-TRN-001 (M1)** 수련: 스승 시범 궤적 가이드 → 굿+ 성공마다 페이드→소거(암기), 굿+ 3연속 통과, 스승 대사 등급 연동.
- **FR-TRI-001 (M1)** 승급 시험(마나 각성): 지정 4획 연속(획 간 ≤2.0s)·평균 70+·미스 즉시 중단. 불합격=최저 정확도 획 지적+재도전.
- **FR-MAP-001 (M2)** 노드맵: 스테이지 3구역, 노드 5종(전투/수련/이벤트/상점/휴식). 결전 15분당 1회 이하(NFR-008).
- **FR-SAV-001 (M2)** 저장(localStorage): 구역 체크포인트 + 노드 완료 이어하기(version 필드). 손실 최대 1노드.
- **FR-ECO-001 (M2)** 경제: 골드·상점(아이템 3+검 3)·레벨(HP+10/위력+2). 아이템은 관찰 페이즈만.
- **FR-STY-001 (M1)** 유파: styles.json + StyleManager. 미러=좌표 기반(원 chirality 정확). 검술 조합은 canonical(입력이 canonical로 접힘) — 미러 유파 오버라이드 불필요.
- **FR-OPS-003 (M2, 승인 대기)** 최소 플레이 로그: 스테이지 이탈 지점 + 획별 평균 정확도만 익명 수집(Supabase). **T2-10 사용자 승인 시에만 구현.**

## 6. 피드백 요구사항 (FR-FBK)
- **FR-FBK-001** 등급별 검명 5종+햅틱 — 화면 안 봐도 등급 인지.
- **FR-FBK-002** 실패 시 내 궤적 vs 이상 궤적 비교 오버레이.
- **FR-FBK-003** 메트로놈(비트 마커, 토글). AudioContext는 사용자 제스처 시 resume.
- **FR-FBK-004 (M1)** 결전 방향 오디오 큐(공격 방향별 톤) — 소리만으로 응수 방향 인지.

## 7. 비기능 요구사항 (NFR)
- **NFR-008** 결전은 15분 세션당 1회 이하 배치(피로도 관리).
- **NFR-011 (개정)** 웹 표준 스택(TypeScript + Vite, DOM + Canvas). **렌더링 프레임워크(Phaser 등)는 필요 시 도입** — M1/M2는 DOM+Canvas 유지. 판정 엔진이 순수 TS로 분리돼 이관 비용 통제됨. 명확한 성능/연출 한계 도달 시 재평가(멈추고 보고).
