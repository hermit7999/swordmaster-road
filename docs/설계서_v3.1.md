## 1. 개요

- 획(劃)을 그어 검을 휘두르는 제스처 검술 액션 RPG. 모바일+PC 웹, 단일 코드베이스, 가로 고정.
- 입력 2방식 병행: **검로(劍路)** 궤적 긋기(정확도=궤적 정밀도) · **검결(劍訣)** 방향 커맨드(정확도=리듬/타이밍).
- 판정 로직은 **순수 함수**(DOM 비의존). 수치는 데이터 블록에만. 판정 이벤트 규격 고정: `{strokeId, accuracy, grade, inputMode, timestamp}`.
- 등급 5단계: 미스 / 배드 / 굿 / 그레이트 / 퍼펙트.

## 2. 판정·입력 시스템

### 2.1 입력 채널 & 중재 (InputArbiter)

두 채널(검로=캔버스 포인터, 검결=키보드/가상패드)이 **동시 활성**된다. 중재는 **행동 점유(occupancy) 모델**:

- 행동을 먼저 시작한 채널이 그 행동이 끝날 때까지 채널을 점유한다.
  - 검로: `pointerdown`에 점유 획득, 판정(`endStroke`) 시 해제.
  - 검결: 버퍼 첫 입력에 점유 획득, 확정(`resolveCommand`) 시 해제(인식 성공/실패 무관).
- 점유 중 다른 채널의 새 행동 시작은 **무시(drop)** 된다. 겹치지 않는(순차) 행동은 채널을 섞어도 모두 발행된다.
- 안전 백스톱 `arbiterStaleMs`(2000ms): 어떤 이유로 해제가 누락되어도 이 시간 뒤 다른 채널이 점유를 넘겨받아 영구 잠금을 방지한다(정상 경로 아님).
- **주의(부채):** 현재는 '무시' 모델. 빠른 결전 응수 페이즈에서 의도 입력이 드롭될 수 있어, M1(T1-07)에서 우선순위/큐/캔슬로 정식화한다.

> M0의 구 `arbiterLockMs`(110ms 시간 기반 blanket 차단)는 빠른 연속 검결의 2번째 발행을 삼키는 결함이 있어 본 점유 모델로 대체되었다.

**검결 커맨드 확정 규칙:**
- 버퍼가 완전한 커맨드 패턴과 **완전 일치**하면 즉시 확정한다(리듬 타이밍은 각 입력 타임스탬프로 이미 보존됨). M0의 3획 커맨드는 서로 접두어 관계가 아니므로 즉시 확정이 안전하다.
- 아직 불완전하면 `commandResolveMs`(640ms) 뒤 확정한다. 640ms = 정박(beatMs 400) + 배드창(280) 여유 — 늦은 2번째 비트를 허용하기 위함(구 260ms는 정박 전 버퍼를 쪼개 정박 플레이를 불가능하게 했음).
- 동시 입력(카디널 쌍)은 `simulMs`(70ms) 이내면 대각 토큰으로 접는다.

### 2.2 검로(劍路) 파이프라인

`judgeStroke(raw, meta, style)` (순수):
1. 유파가 좌수면 입력 x를 미러(정규=우수 공간으로 접기).
2. 탭/미세 입력 거부: bbox대각/캔버스대각 < `rejectRatio`(0.035) → 탭. 경로길이/캔버스대각 < `minLenRatio`(0.12, 획별 오버라이드 가능) → 짧음.
3. 64포인트 리샘플 → **크기 정규화(회전 비정규화 — 방향이 획의 정체성)**.
4. 템플릿 비교(평균 포인트 거리) 최근접 획 식별. 최소거리 > `classifyThreshold`(0.34) → 획 불명.
5. 정확도 = 방향 40% + 직선 30% + 속도 15% + 완주 15%.

### 2.3 점수 산식 **(v3.1 변경)**

| 성분 | 가중 | 산식 | 비고 |
|---|---|---|---|
| **방향** | 40% | 리샘플된 **각 구간(segment)** 단위벡터와 템플릿 방향의 내적을 clamp(0..) 평균 | **v3.0의 "순(net) 시작→끝 벡터"에서 변경.** 순 벡터는 좌→우로 흔들려 그어도 시작·끝점만 맞으면 방향 만점 → 방향40+속도15+완주15=70점 바닥이 보장되어 "흔들린 획"이 배드(50–69)에 도달 불가했다. 국소 방향 일치도는 흔들리면 하락하여 "정확도=궤적 정밀도" 취지를 만족. |
| **직선** | 30% | (정규화 현/정규화 경로길이)**²** | 제곱으로 흔들림 감점을 강화. (v3.0 미명시분 확정) |
| **속도** | 15% | 이상 구간 `120~700ms` 내 1, 밖은 선형 감쇠 | |
| **완주** | 15% | 정규화 끝점 정합도(1 − 시작·끝점 오차 평균) | 작게 그어도 비율 맞으면 인정 |

검증(합성): 흔들림 강도(jitter)에 따라 그레이트→굿→배드로 연속 감점(20/40/60/80/100 = 91/73/60/54/50점). 등급컷·가중치는 불변.

### 2.4 검결(劍訣) 리듬 판정 (RhythmJudge) & 수치표

`judgeRhythm(rawInputs, strokeId, style)` (순수): 이상 비트 타임라인 대비 타이밍 오차.
- 판정창: 퍼펙트 ±50 / 그레이트 ±100 / 굿 ±180 / 배드 ±280ms. 순서·개수 오류 = 미스.
- 자기참조(self-referenced): 첫 입력 = 다운비트, 이후 입력 오프셋을 이상 오프셋과 비교(외부 클럭 미동기 — §2 결정 유지, 수련 메트로놈만 연습 UI로 외부 클럭 제공).
- **위력 +10% 조건**: `raw 입력 ≥ 2` **&&** 등급 ≠ 미스 **&&** 최대 타이밍오차 ≤ 100ms(그레이트 창). **패드 단일 대각(예: ↘ 한 번) press는 제외**(다입력이 아니므로).

**BALANCE 수치표 (M0 확정):**

| 키 | 값 | 설명 | 출처 |
|---|---|---|---|
| resample | 64 | 리샘플 포인트 | 원안 |

**획 데이터(STROKE_TEMPLATES)** 필드: `path`(이상 궤적), `command`(검결 토큰), **`raw`**(이상 raw 입력 토큰; 패드 대각 단일 → 카디널 2개 전개), **`rhythm`**(각 raw 토큰의 이상 오프셋 ms). M0 3종: 횡베기 rhythm[0,400] / 사선 [0,0](동시) / 내려찍기 [0,400].

### 2.5 유파(流派)

`STYLES`: `mirrorX`(좌우 미러) + `updownCluster`(가상패드 ↑↓ 위치, 우수=right/좌수=left). 미러는 판정·커맨드 양쪽에서 입력을 정규(우수) 공간으로 접어 처리 → 품질 동등(우수=좌수 accuracy 동일, 검증됨). M1에서 획ID 미러 매핑 테이블(h_lr↔h_rl, diag_dr↔diag_dl)로 단순화 예정.

## 3. 콤보/마나 (요약)

`ComboTracker`: 검술 조합을 획 이벤트 시퀀스로 추적. 미스 한 획 = 조합 끊김. 성립 시 마나 = `cost × (2 − 평균정확도/100)`, 평균 ≥ 95면 위력 +20%. M0: 연풍참(횡베기→사선↘, cost 15, 창 1.5s). 검결 +10%는 M1(T1-05)에서 실데미지 곱셈에 연결.

## 4. 변경 이력

| 버전 | 변경 |
|---|---|
| v3.0 → **v3.1** | 방향 점수 = 국소 방향 일치도(구 순 벡터). 직선도 = (현/경로)². 커맨드 확정 = 완전일치 즉시확정 + 640ms. 중재 = occupancy 모델(+staleMs 2000, 구 arbiterLockMs 폐기). 신규 수치 simulMs/commandWindow/commandResolveMs/arbiterStaleMs/beatMs 등재. 획 데이터에 raw/rhythm 필드 추가. 위력+10% 조건 명문화. 제스처 좌표 clientX−rect. |
