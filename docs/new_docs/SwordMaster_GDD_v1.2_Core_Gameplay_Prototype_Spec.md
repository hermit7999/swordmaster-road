# SwordMaster GDD v1.2

# Core Gameplay Prototype Spec

## 목적

이 문서는 게임을 완성하기 위한 문서가 아니다.

**'핵심 재미(Core Fun)'를 가장 빨리 검증하기 위한 문서**이다.

------------------------------------------------------------------------

# 왜 필요한가?

5분짜리 스테이지를 만드는 데 2개월이 걸릴 수 있다.

만약 재미가 없다면 2개월을 잃는다.

따라서 먼저 3초 → 30초 → 60초 → 5분 순서로 핵심 재미를 검증한다.

------------------------------------------------------------------------

# Prototype Layer

## Layer 1 : 첫 3초

질문

"검을 처음 그은 순간 손맛이 있는가?"

검증 항목

-   검 궤적
-   타격음
-   히트스톱
-   화면 흔들림
-   피격 연출

성공 기준

플레이어가

"오? 베는 맛 좋은데."

라고 느낀다.

------------------------------------------------------------------------

## Layer 2 : 첫 30초

질문

"계속 적을 베고 싶은가?"

구성

-   일반 병사 3명
-   발도술 획득
-   일반 병사 3명
-   작은 정예 몬스터

검증

-   이동이 재미있는가?
-   검술 입력이 자연스러운가?
-   적을 계속 찾게 되는가?

------------------------------------------------------------------------

## Layer 3 : 첫 60초

질문

"성장한 것이 체감되는가?"

추가

-   첫 성장 선택
-   첫 빌드 변화
-   첫 작은 위험

성공 기준

"아까보다 훨씬 강해졌다."

가 아니라

"플레이 방식이 달라졌다."

를 느껴야 한다.

------------------------------------------------------------------------

## Layer 4 : 첫 5분

질문

"한 판만 더 하고 싶은가?"

구성

-   일반전
-   성장 선택
-   미니보스
-   Duel Mode 보스

성공 기준

보스를 잡은 뒤

"다른 빌드도 해보고 싶다."

라는 생각이 든다.

------------------------------------------------------------------------

# Prototype 성공 조건

□ 첫 3초가 강렬하다.

□ 첫 30초가 지루하지 않다.

□ 첫 60초에 성장이 체감된다.

□ 첫 5분이 짧게 느껴진다.

□ 플레이어가 다시 시작한다.

------------------------------------------------------------------------

# 구현 순서

1.  검 타격감
2.  이동
3.  일반 몬스터
4.  검술 3개
5.  성장 선택 3개
6.  미니보스
7.  Duel Mode

기능을 늘리지 않는다.

재미가 확인되면 다음 단계로 넘어간다.

------------------------------------------------------------------------

# 앞으로 개발 원칙

새로운 기능을 추가하기 전에 반드시 질문한다.

1.  이 기능이 첫 5분을 더 재미있게 만드는가?
2.  1인 개발자가 2\~4주 안에 만들 수 있는가?
3.  이 기능이 게임의 핵심(검술)에 집중하는가?

YES가 아니면 보류한다.
