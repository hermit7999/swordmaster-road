# 1. 목표 수치 (설계 계약 — Difficulty/Core Loop Bible 승계)

| 항목 | 목표 |
|---|---|
| 스테이지 성공률 | S1 90% / S2 80% / S3 70% / S4 60% / S5 첫 30~40%·재도전 70%+ |
| 보스 시도 횟수 | B1 1~2회 / B2~4 2~4회 / B5 3~6회 |
| 잡몹 처치 시간 | 5~10초 (1~2획) |
| 정예 처치 시간 | 1~2분 |
| 보스전 시간 | 3~5분 |
| 한 판 완주 | 30~40분 |
| 성장 선택 체감 | 60초 내 "플레이 방식 변화" 인지 |
| 첫 이탈 지점 | 첫 5분 이탈률 최소화 (텔레메트리 추적) |

# 2. balance.json 구조 (전 패키지 수치의 단일 출처)

```
balance.json
├── player   (HP, 기력, 회복, 회피, i-frame)
├── grades   (등급 경계·배율)
├── skills   (per-skill 오버라이드)
├── ki       (kiPerHit, kiMax, 피격 감소율)
├── combo    (배율, 리셋 시간)
├── poise    (그로기 시간, 자연 감소)
├── enemies  (종별 HP/poise/피해 스케일)
├── bosses   (보스별 스케일, phase 임계)
├── duel     (공격권 게이지, 패링 창, 예고 최소시간)
├── stages   (웨이브 밀도, 동시 상한)
├── economy  (골드 곡선, 가격, 드랍률)
├── growth   (희귀도 확률, 선택 횟수)
├── difficulty (관대/표준/도전 배율 세트)
└── telemetry_targets (§1의 목표 수치 — 대시보드 비교용)
```

# 3. 피해 체인 역산 절차 (절대값 산정법)

1. 목표에서 역산: "잡몹 = 굿 등급 2획" → soldier HP = slash_h 기본 20 × 1.0 × 2 = 40 근처 → 22~40 밴드.
2. 보스: "3~5분 × 평균 DPS" 역산 + 자세 붕괴 2~3회 경로 병행.
3. 플레이어 피격: 잡몹 3~4대 / 보스 필살 외 최대 40% (P03 §8).
4. 전 수치 변경은 이 절차의 재계산으로 — 개별 손튜닝 지양.

# 4. 난이도 3단 (설정 연동 — UI-06)

|---|---|---|---|

# 5. 튜닝 절차 (반복 프로세스)

1. 텔레메트리 수집 (P14): 스테이지 이탈 지점, 획별 성공률/오인식, 보스 시도 수, 스킬 사용 분포, 사인 통계.
2. 주간 리뷰: telemetry_targets 대비 편차 → balance.json 수정안.
3. 시뮬 검증: 봇 플레이 100런 회귀 (성공률·완주 시간 밴드).
4. 배포: 데이터만 패치 (P15 핫픽스 경로).

# 7. Acceptance Test

□ 하드코딩 수치 0 (정적 린트 — 수치는 전부 data/)
□ 봇 시뮬: §1 목표 밴드 내 (성공률 ±10%p, 시간 ±20%)
□ 사장 스킬 없음 (사용률 5% 미만 스킬 0)
□ 난이도 3단: 관대에서 S1 95%+, 도전에서 S5 유의미

□ balance.json만 수정해 잡몹 HP 2배 → 재빌드 없이 반영 (핫 리로드 or 재시작 반영)
