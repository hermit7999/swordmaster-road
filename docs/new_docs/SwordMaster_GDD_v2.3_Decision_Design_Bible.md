# SwordMaster GDD v2.3

# Decision Design Bible

> 목적
>
> 이 문서는 플레이어가 게임에서 내리는 모든 선택을 설계한다. 좋은 게임은
> 선택지가 많은 게임이 아니라, **의미 있는 선택이 많은 게임**이다.

------------------------------------------------------------------------

# 1. 핵심 철학

플레이어는 항상 고민해야 한다.

하지만 답이 하나뿐인 선택은 만들지 않는다.

좋은 선택 A

VS

좋은 선택 B

가 되어야 한다.

------------------------------------------------------------------------

# 2. 선택의 종류

## 전투 선택

-   공격할까?
-   회피할까?
-   패링할까?
-   기력을 아낄까?

------------------------------------------------------------------------

## 성장 선택

-   기존 검술 강화
-   새로운 검술 습득
-   패시브 선택
-   기력/생존 강화

------------------------------------------------------------------------

## 탐험 선택

-   안전한 길
-   위험하지만 보상이 큰 길

------------------------------------------------------------------------

## 보스 선택

-   지금 도전
-   더 성장 후 도전

------------------------------------------------------------------------

# 3. 좋은 선택의 조건

□ 각각 장점이 있다.

□ 정답이 없다.

□ 플레이 스타일이 달라진다.

□ 다음 플레이에서 다른 선택을 해보고 싶어진다.

------------------------------------------------------------------------

# 4. 절대 금지

-   무조건 정답인 선택
-   함정 선택
-   설명 없는 선택
-   숫자만 큰 선택

------------------------------------------------------------------------

# 5. 로그라이크 선택 규칙

보상은 항상 3개 제시한다.

예)

1.  발도술 중심
2.  패링 중심
3.  생존 중심

세 가지 모두 끝까지 플레이 가능해야 한다.

------------------------------------------------------------------------

# 6. 리스크 vs 리워드

위험이 크면 보상도 커야 한다.

안전한 선택은 안정적이지만 폭발력이 낮다.

------------------------------------------------------------------------

# 7. 플레이어 심리

선택 후

"아쉽지만 다른 것도 재미있었겠다."

를 목표로 한다.

"잘못 골랐다."

를 느끼게 하지 않는다.

------------------------------------------------------------------------

# 8. 체크리스트

새로운 선택지를 만들 때

□ 빌드가 달라지는가? □ 재미가 달라지는가? □ 다른 플레이어도 다른 선택을
할 수 있는가? □ 실력을 완전히 무시하지 않는가?
