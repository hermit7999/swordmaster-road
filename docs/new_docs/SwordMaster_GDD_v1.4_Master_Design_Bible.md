# SwordMaster GDD v1.4

# Master Design Bible (프로젝트 헌법)

> 이 문서는 프로젝트 전체에서 절대 쉽게 바뀌면 안 되는 원칙을 정의한다.
> 이후 작성되는 모든 GDD, Combat Bible, Boss Bible, UI Bible, AI 개발
> 문서는 이 문서를 기준으로 판단한다.

------------------------------------------------------------------------

# 1. 프로젝트 미션

**1인 개발자가 완성할 수 있는 최고의 검술 액션 게임을 만든다.**

목표는 AAA 규모가 아니라, 플레이어가 "검을 직접 휘두르는 재미"를 느끼게
하는 것이다.

------------------------------------------------------------------------

# 2. Core Identity

이 게임은 다음 다섯 가지로 정의된다.

1.  손으로 직접 검술을 그린다.
2.  필드는 빠르고 시원하다.
3.  보스는 긴장감 있는 Duel Mode다.
4.  실력이 승패를 결정한다.
5.  성장은 실력을 보완한다.

이 다섯 가지를 해치는 기능은 추가하지 않는다.

------------------------------------------------------------------------

# 3. 디자인 원칙

## Rule 1 : 검술이 가장 중요하다.

모든 기능은 검술을 더 재미있게 만들어야 한다.

------------------------------------------------------------------------

## Rule 2 : 손맛이 그래픽보다 중요하다.

그래픽은 좋아질 수 있다.

하지만 손맛은 처음부터 좋아야 한다.

------------------------------------------------------------------------

## Rule 3 : 기능보다 완성도를 우선한다.

기능 100개보다 완성도 높은 기능 10개가 더 중요하다.

------------------------------------------------------------------------

## Rule 4 : 1인 개발 범위를 절대 넘지 않는다.

새 기능을 추가하기 전에

-   2\~4주 안에 구현 가능한가?
-   유지보수 가능한가?

를 반드시 검토한다.

------------------------------------------------------------------------

## Rule 5 : 실력이 가장 중요하다.

플레이어는

"아이템이 좋아서 이겼다."

가 아니라

"내 컨트롤이 좋아져서 이겼다."

를 느껴야 한다.

------------------------------------------------------------------------

## Rule 6 : 성장은 실력을 보완한다.

영구 성장과 장비는 실력을 대체하지 않는다.

실력이 있는 플레이어를 더욱 빛나게 만든다.

------------------------------------------------------------------------

## Rule 7 : 모든 적은 선생님이다.

새로운 적은 새로운 기술을 배우게 만들어야 한다.

보스는 배운 것을 시험한다.

------------------------------------------------------------------------

## Rule 8 : 모든 스테이지는 학습 곡선이다.

배움 ↓

연습 ↓

응용 ↓

시험 ↓

보상

이 구조를 유지한다.

------------------------------------------------------------------------

## Rule 9 : 재미가 검증되지 않은 기능은 구현하지 않는다.

모든 기능은

-   Fun Checklist
-   Feature Priority Matrix

를 통과해야 한다.

------------------------------------------------------------------------

## Rule 10 : 출시 후 확장을 고려한다.

출시 버전은 작게, 확장은 크게.

DLC와 업데이트를 전제로 설계한다.

------------------------------------------------------------------------

# 5. 성공 기준

플레이어가 아래 네 문장을 말하면 성공이다.

-   "베는 맛이 좋다."
-   "내 컨트롤이 늘었다."
-   "이번 빌드 재미있다."
-   "한 판만 더."
