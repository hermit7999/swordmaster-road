# SwordMaster GDD v0.7

# MVP Definition (최소 출시 범위)

## 왜 MVP가 필요한가?

이 프로젝트는 **1인 개발**을 전제로 한다.

따라서 좋은 아이디어를 모두 구현하는 것이 아니라, 가장 재미있는 경험만
출시 버전에 포함한다.

------------------------------------------------------------------------

# 개발 원칙

## 반드시 지킨다

-   재미가 최우선
-   1년 내 출시 가능
-   시스템은 적고 깊게
-   반복 플레이가 가능해야 함

## 하지 않는다

-   오픈월드
-   수십 개의 NPC
-   복잡한 퀘스트
-   장비 파밍 게임
-   MMORPG 수준의 성장
-   수백 개의 몬스터

------------------------------------------------------------------------

# MVP 목표

플레이어가

> "5분만 하려고 했는데 1시간이 지나 있었다."

라고 말하는 게임.

------------------------------------------------------------------------

# 출시 버전에 반드시 포함

## 필드

-   작은 맵 3개
-   일반 몬스터 8\~10종
-   정예 몬스터 3종

------------------------------------------------------------------------

## 전투

-   왼손 이동
-   오른손 제스처 검술
-   검술 6\~8개

------------------------------------------------------------------------

## 보스

-   Duel Mode
-   보스 3명
-   보스당 3\~5개의 핵심 패턴

------------------------------------------------------------------------

## 성장

-   검술 진화
-   로그라이크 선택
-   영구 성장
-   검파 3개

------------------------------------------------------------------------

## 콘텐츠

-   플레이 타임 : 30\~40분
-   반복 플레이 가능
-   엔딩 1개

------------------------------------------------------------------------

# 성공 기준

## 플레이 테스트

-   첫 5분 이탈률이 낮다.
-   첫 보스를 다시 도전하고 싶다.
-   새로운 빌드를 시험하고 싶다.
-   '한 판만 더'가 자연스럽게 나온다.

## Steam

목표

-   긍정적 평가 90% 이상
-   플레이타임 평균 10시간 이상
-   DLC 확장 가능한 구조
