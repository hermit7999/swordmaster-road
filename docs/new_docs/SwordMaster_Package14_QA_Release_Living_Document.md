# 1. CI 게이트 (전 커밋)

```
push → tsc --noEmit → vitest (전 유닛+Acceptance 자동분) → build → (main) 배포
```
- 실패 시 머지 금지. Acceptance 자동 TC는 패키지별 태그로 분리 실행 가능.
- 리플레이 회귀: 녹화된 입력 시퀀스 세트(황금 리플레이) → 판정/전투 결과 비교 (결정론 활용).

# 3. 텔레메트리 (익명, 옵트아웃 제공)

- 이벤트: 세션 시작/종료, 스테이지 진입/이탈/사망(사인), 보스 시도/결과, 획별 성공·오인식률, 스킬 사용 분포, 성장 선택 픽률, FPS 분포.
- 개인정보 금지 (Input Spec §20). 좌표 원본 로컬 보관 원칙.
- 백엔드: 경량 수집(예: Supabase) — 프로토타입 T2-10 계획 승계. 대시보드: telemetry_targets(P13) 대비 자동 비교.

# 4. 배포 채널 파이프라인

| 채널 | 방법 | 시점 |
|---|---|---|
| 웹 (데모) | GitHub Pages — push 자동 배포 (프로토타입 워크플로 재사용) | 상시 (플레이테스트) |
| Steam | Tauri 패키징 + Steamworks (업적/클라우드) | 데모 검증 후 |
| 모바일 | Capacitor + 스토어 (IAP 훅) | Steam 후 |

- 버전 규칙: semver + 빌드 태그. 데이터 핫픽스는 patch.

# 5. 릴리스 체크리스트 (요약 — P16에서 최종 게이트)

□ 전 패키지 Acceptance PASS
□ 황금 리플레이 회귀 PASS
□ 기기 매트릭스 실기 완주 (각 1회+)
□ 30~40분 완주 시간 실측
□ 저장 손상 시나리오 통과
□ 성능: 폰 60FPS / 메모리 안정 (30분)
□ 텔레메트리 수신 확인
□ 스토어 자산 (스크린샷/트레일러 — 5초 안에 차별점 전달)
□ 크레딧/라이선스 고지 (아트 모델 포함)
