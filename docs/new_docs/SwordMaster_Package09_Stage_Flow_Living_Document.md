# Package 09 — Stage Flow (Living Document) v1.0

> 작성일: 2026-07-12 | 선행: Package 04, 06, 08 | 해금: Package 10, 12, 13
> 5분 스테이지 × 5 = 한 판 30~40분의 실제 흐름을 정의한다.
> **채택 결정(ADR-012 후보)**: 필드 = 횡스크롤 사이드뷰 진행형 (프로토타입 아케이드 스파이크 검증 자산 — "달리며 조우 즉시 베기", 씬 전환 없음).

---

# 1. 목적/범위

포함: 스테이지 구조, 웨이브/스폰 규칙, 노드(구간) 타입, 진행 리듬, 실패/재시작, stage.json.
제외: 개별 적 행동(P04), 보스 내용(P06), 연출(P12).

# 2. 스테이지 기본 구조 (1스테이지 ≈ 5~7분)

```
[진입 연출] → 구간A(학살 웨이브) → 구간B(휴지: 상점/휴식/이벤트 중 1)
→ 구간C(긴장 웨이브 + 정예) → 성장 선택 → 구간D(폭발 웨이브)
→ 미니보스(짝수 스테이지) → [보스 관문] → Duel → 보상 → 다음 스테이지
```
- 템포 파형: 학살→휴지→긴장→폭발 (프로토타입 검증 리듬).
- 성장 선택: 스테이지당 2회(웨이브 보상 1 + 정예 1) + 보스 1회.

# 3. 스테이지 5개 테마 (Stage Design Bible 매핑)

| # | 테마(가칭) | 핵심 학습 | 신규 적 | 보스 |
|---|---|---|---|---|
| S1 | 산길 — 검을 배우다 | 이동·기본베기·발도술 | soldier, spear, hound | M1→B1 노병 |
| S2 | 대나무 숲 — 검을 익히다 | 회피·거리 | archer, swift, shield | M2→B2 무희 |
| S3 | 폐성 — 검을 연결하다 | 연계·오의 | dual, heavy | M3→B3 수문장 |
| S4 | 설원 — 검을 응용하다 | 빌드 활용 | berserk, caster, shadow | M4→B4 그림자 |
| S5 | 검성봉 — 검성의 시험 | 종합 (신규 없음) | knight + 혼합 | M5→B5 검성 |

- 교사 선행 규칙: 보스가 시험할 기술의 교사 적이 해당 스테이지 구간A~C에 반드시 등장 (검증 규칙).

# 4. 웨이브/스폰 규칙

- 웨이브 = spawn_table 기반: {enemy_id, count, 간격, 방향(전/후), 트리거(진입 x좌표|전멸)}.
- 동시 활성 적 상한 6 (성능·가독성, balance). 빨강 예고 동시 1 (P04 §6).
- 웨이브 전멸 → 무리 피니시 연출 트리거 (P12 — 슬로우+줌, 프로토타입 검증).
- 후방 스폰은 경고 후에만 (억울한 배후 피격 금지).
- 구간 사이 "달리기 구간" 확보 — 전투 없는 15~30초 (호흡, Flow 관리).

# 5. 휴지 구간 (구간B) 노드 3종 중 랜덤 1

| 타입 | 내용 |
|---|---|
| 상점 | 골드 사용 (P08 §4) |
| 휴식 | HP 회복 40% + 다음 웨이브 미리보기 |
| 이벤트 | 텍스트 선택 이벤트 (리스크 vs 리워드 — Decision Bible. 예: "낡은 사당: 기도한다(패시브)/부순다(골드+저주)") |

# 6. 실패/재시작

- 사망 → 사인 표시 → **스테이지 시작 체크포인트** 부활 (런 유지, 스테이지 내 진행만 초기화).
- 체크포인트 부활 무제한, 단 스테이지 재시작 시 웨이브 시드 재추첨 (암기 방지·신선도).
- 런 포기(메뉴): 결과 화면 → 수련점 정산.
- 보스전 사망도 동일 (보스 앞이 아닌 스테이지 시작 — 단, S5만 보스 관문 체크포인트 제공).

# 9. Acceptance Test

□ 교사 선행 규칙: 5스테이지 전부 데이터 린트 PASS
□ 동시 적 상한·빨강 1개·후방 경고 규칙 위반 0
□ 템포 파형: 구간별 전투 밀도 곡선이 설계(학살>휴지<긴장<폭발) 일치
□ 체크포인트: 부활 후 런 빌드 유지·스테이지 진행 초기화 정확
□ 시드 재추첨: 재시작 시 웨이브 구성 변화 확인
□ 완주 시간: S1 시뮬 5~7분 / 전체 30~40분 (봇 기준)
□ 실기: 5분이 짧게 느껴지는가 (테스터 질문)
