# SwordMaster GDD v1.1

# Stage Design Bible (초안)

## 목적

스테이지는 단순한 맵이 아니다.

스테이지는 플레이어에게 검술을 가르치고, 연습시키고, 시험하는 과정이다.

------------------------------------------------------------------------

# 핵심 공식

배움 ↓

연습 ↓

응용 ↓

시험 ↓

보상

이 공식을 모든 스테이지가 반복한다.

------------------------------------------------------------------------

# 전체 게임 구조

Stage 1 : 검을 배우다 Stage 2 : 검을 익히다 Stage 3 : 검을 연결하다
Stage 4 : 검을 응용하다 Stage 5 : 검성의 시험

약 30\~40분

------------------------------------------------------------------------

# Stage 1

## 목표

플레이어가 게임 조작을 자연스럽게 익힌다.

### 배우는 것

-   이동
-   기본 베기
-   첫 검술(발도술)

### 적

-   일반 병사
-   느린 창병

### 미니보스

발도술을 사용하면 쉽게 이길 수 있다.

### 보스

패턴 3개

시험

-   이동
-   기본베기
-   발도술

------------------------------------------------------------------------

# Stage 2

## 목표

거리와 회피

### 배우는 것

-   회피
-   거리 유지
-   연계 시작

### 적

-   궁수
-   빠른 검객

### 보스

회피와 발도술을 함께 요구

------------------------------------------------------------------------

# Stage 3

## 목표

콤보

### 배우는 것

-   검술 연계
-   첫 오의

### 적

-   방패병
-   쌍검

### 보스

패턴을 읽고 콤보 성공

------------------------------------------------------------------------

# Stage 4

## 목표

빌드의 차이 체험

플레이어마다 사용하는 검술이 달라진다.

보스는 다양한 해결 방법을 허용한다.

------------------------------------------------------------------------

# Stage 5

## 목표

최종 시험

새로운 기술은 없다.

지금까지 익힌 모든 것을 활용한다.

최종 Duel Mode

------------------------------------------------------------------------

# 스테이지 설계 원칙

## 한 스테이지 = 한 가지 핵심 학습

Stage1 : 기본베기

Stage2 : 회피

Stage3 : 연계

Stage4 : 빌드

Stage5 : 종합

새로운 요소를 여러 개 동시에 넣지 않는다.

------------------------------------------------------------------------

# 체크리스트

모든 스테이지는 답할 수 있어야 한다.

1.  무엇을 배우는가?
2.  무엇을 연습하는가?
3.  무엇을 시험하는가?
4.  무엇을 보상으로 주는가?
5.  다음 스테이지가 기대되는가?

답이 없다면 재설계한다.
