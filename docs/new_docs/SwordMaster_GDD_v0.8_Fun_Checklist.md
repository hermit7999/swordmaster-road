# Fun Checklist & Design Validation

## 목적

이 문서는 새로운 기능을 추가하기 위한 문서가 아니다.

반대로

**'이 기능을 정말 넣어야 하는가?'**

를 검증하기 위한 문서이다.

# 개발 원칙

# Fun Checklist

## 1. 손맛이 좋아지는가? ⭐⭐⭐⭐⭐

질문

-   검을 그었을 때 더 시원한가?
-   타격감이 좋아졌는가?
-   제스처를 사용할 이유가 생겼는가?

YES / NO

------------------------------------------------------------------------

## 2. 플레이어의 실력이 중요해지는가? ⭐⭐⭐⭐⭐

질문

-   잘하는 사람은 더 잘할 수 있는가?
-   단순 스탯으로 해결되지 않는가?
-   컨트롤이 승패에 영향을 주는가?

YES / NO

------------------------------------------------------------------------

## 3. 성장의 의미가 있는가? ⭐⭐⭐⭐☆

질문

-   플레이 방식이 바뀌는가?
-   새로운 선택지가 생기는가?
-   같은 플레이를 반복하지 않는가?

YES / NO

------------------------------------------------------------------------

## 4. 1인 개발 가능한가? ⭐⭐⭐⭐⭐

질문

-   2\~4주 안에 구현 가능한가?
-   유지보수가 쉬운가?
-   AI 에셋을 활용할 수 있는가?

YES / NO

------------------------------------------------------------------------

## 5. Steam에서 차별점이 되는가? ⭐⭐⭐⭐⭐

질문

-   다른 게임에도 있는 기능인가?
-   이 게임만의 특징인가?
-   트레일러에서 5초 안에 전달되는가?

YES / NO

------------------------------------------------------------------------

# 기능 추가 기준

5개 중

-   5 YES → 바로 구현 후보
-   4 YES → 구현 고려
-   3 이하 → 보류
