# Package 15 — Live Operation

## 1. 패치 정책

| 종류 | 내용 | 주기 |
|---|---|---|
| 핫픽스 | balance.json/데이터만 — 코드 무변경 | 수시 (당일 가능) |
| 패치 | 버그 수정 + 소규모 개선 | 2주 |
| 업데이트 | B티어 콘텐츠 (신규 검술·챌린지·보스러시) | 분기 |
| DLC | C티어 (신규 지역·검파·스토리) | 출시 후 로드맵 |

## 2. 운영 규칙

- 밸런스 변경은 반드시: 텔레메트리 근거 → 시뮬 검증(BAL-02) → 패치노트 명시. "몰래 너프" 금지.
- Save 호환: 전 패치에 마이그레이션 경로 (SAV-05). 세이브 파손 버그 = 최우선 등급.
- DLC 호환: 콘텐츠는 ID 기반 추가만 — 기존 ID 변경 금지 (Data Architecture §2).
- 피드백 채널: 게임 내 버튼(타이틀) + 커뮤니티. 이슈 트리아지 주간.

# Package 16 — Final Release Review

## 1. 설계 Freeze Gate (전 게이트 요약)

| Gate | 대상 | 조건 |
|---|---|---|
| A — Combat Core | P01~03 | 입력·제스처·스킬·전투 Acceptance 전체 정의·통과 |
| B — Content Core | P04~06 | 적·Duel·보스 상태머신+Schema+시나리오 통과 |
| C — Game Loop | P07~09 | 성장·장비·스테이지 연결 규칙 + 100런 시뮬 |
| D — Product | P10~15 | UI·Save·연출·밸런스·QA·운영 전체 |

## 2. Release Gate Checklist (최종)

□ 4 Gate 전부 통과
□ 미설계 항목 0 / 문서 충돌 0 / Open Question 전부 종결 or 명시적 이월
□ MVP 체크리스트 (마스터 문서 §12.7) 전 항목
□ 성공 기준 실측: 테스터 "한 판만 더" 발화 / 첫 5분 이탈률 / 평균 세션
□ 스토어 준비물 완비 + 가격 확정
□ 리스크 등록부 최종 리뷰 (심각 리스크 0)
□ 출시 후 2주 핫픽스 대기 체제

## 3. Freeze 선언 절차

1. 전 패키지 Living Document 최종판 태그
2. 이 문서에 Freeze 선언 기록 (일자·버전·서명)
3. 이후 변경은 전부 ADR + P15 절차로만
