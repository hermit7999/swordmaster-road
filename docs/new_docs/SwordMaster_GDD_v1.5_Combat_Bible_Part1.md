# SwordMaster GDD v1.5

# Combat Bible (Part 1)

> 이 문서는 전투의 '손맛'을 규칙으로 정의한다. 모든 수치는 초기값이며
> 플레이 테스트를 통해 조정한다.

------------------------------------------------------------------------

# 1. 전투 철학

전투의 목표는 적을 많이 죽이는 것이 아니다.

플레이어가

"내 컨트롤이 좋아서 이겼다."

라고 느끼게 만드는 것이다.

------------------------------------------------------------------------

# 2. 전투 우선순위

1.  손맛
2.  가독성
3.  컨트롤
4.  성장
5.  연출

그래픽보다 손맛이 우선이다.

------------------------------------------------------------------------

# 3. 기본 전투 루프

적 발견 ↓

접근

↓

검술 사용

↓

적 반응

↓

보상

↓

다음 적

이 루프는 5\~15초마다 반복된다.

------------------------------------------------------------------------

# 4. 입력 규칙

## 이동

-   왼손 조이스틱
-   항상 이동 가능

## 공격

-   오른손 제스처
-   입력이 끝난 순간 판정 발생

입력은 복잡하지 않아야 한다.

------------------------------------------------------------------------

# 5. 공격 규칙

공격은 반드시 세 가지를 만족해야 한다.

□ 눈으로 보인다.

□ 손으로 느껴진다.

□ 결과가 즉시 나온다.

------------------------------------------------------------------------

# 6. 히트 피드백

공격 성공 시

-   적 경직
-   타격음
-   화면 흔들림
-   히트스톱
-   검 이펙트

최소 3가지 이상이 동시에 발생한다.

------------------------------------------------------------------------

# 7. 회피

목적

공격을 피하는 것이 아니라

'좋은 위치를 만드는 것'

회피는 짧고 즉각적이어야 한다.

------------------------------------------------------------------------

# 8. 패링

패링은 최고 난이도 기술이다.

성공하면

-   큰 손맛
-   기력 회복
-   반격 기회

실패하면

-   큰 패널티가 아니라
-   위치를 잃는다.

------------------------------------------------------------------------

# 9. 그로기

그로기는

많이 때려서가 아니라

정확하게 공략해서 발생한다.

예)

-   약점 공략
-   연속 성공
-   패링 후 반격

------------------------------------------------------------------------

# 10. Duel Mode

일반전과 완전히 다르다.

필드

빠른 학살

↓

Duel

긴장감

↓

필드

속도감

이 리듬을 유지한다.

------------------------------------------------------------------------

# 11. 절대 금지

-   피격 이유가 불명확한 공격
-   보이지 않는 판정
-   과도한 랜덤성
-   HP만 많은 적
-   오래 때려야 하는 적

------------------------------------------------------------------------

# 12. 플레이 테스트 질문

매 빌드마다 아래를 체크한다.

1.  첫 베기가 시원한가?
2.  적을 계속 찾게 되는가?
3.  회피를 쓰고 싶은가?
4.  패링 성공이 짜릿한가?
5.  보스를 다시 도전하고 싶은가?

3개 이하 YES면 전투를 재설계한다.
