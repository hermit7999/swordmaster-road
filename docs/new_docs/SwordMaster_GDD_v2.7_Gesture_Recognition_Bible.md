# SwordMaster GDD v2.7

# Gesture Recognition Bible

> 목적
>
> 이 문서는 제스처 입력을 일관된 규칙으로 인식하기 위한 기술 설계
> 문서이다. 구현자와 AI가 동일한 기준으로 입력을 처리할 수 있도록 한다.

------------------------------------------------------------------------

# 1. 핵심 원칙

-   입력은 플레이어에게 관대하고 결과는 명확해야 한다.
-   잘 그린 입력은 반드시 성공한다.
-   실패 원인을 플레이어가 이해할 수 있어야 한다.
-   기기 성능과 해상도 차이의 영향을 최소화한다.

------------------------------------------------------------------------

# 2. 입력 파이프라인

Touch Down ↓

좌표 수집

↓

좌표 정규화

↓

노이즈 제거

↓

방향 벡터 생성

↓

패턴 매칭

↓

스킬 후보 추출

↓

충돌 해결

↓

스킬 실행

------------------------------------------------------------------------

# 3. 좌표 수집

수집 데이터

-   x
-   y
-   timestamp
-   touch_id

최소 샘플 개수

-   12 Point

최대 샘플 간격

-   16ms 권장
-   33ms 허용

------------------------------------------------------------------------

# 4. 정규화

모든 입력은

-   화면 해상도
-   DPI
-   화면 비율

과 관계없이 동일한 좌표계로 변환한다.

정규화 이후에만 패턴을 비교한다.

------------------------------------------------------------------------

# 5. 제스처 정의

각 제스처는 아래 정보를 가진다.

-   gesture_id
-   name
-   anchor_count
-   direction_sequence
-   minimum_length
-   maximum_length
-   minimum_time
-   maximum_time
-   difficulty
-   unlock_condition

------------------------------------------------------------------------

# 6. 허용 오차

초기 기준

-   방향 오차 : ±20°
-   길이 오차 : ±15%
-   시간 오차 : ±25%

난이도에 따라 조정 가능하다.

------------------------------------------------------------------------

# 7. 입력 시간

너무 짧으면

→ 오입력

너무 길면

→ 실패

판정은 시간보다 패턴 정확도를 우선한다.

------------------------------------------------------------------------

# 8. 후보 충돌

두 개 이상의 스킬이 동시에 일치하면

우선순위

1.  정확도
2.  스킬 레벨
3.  입력 길이
4.  최근 사용 여부

------------------------------------------------------------------------

# 9. 실패 처리

실패 시

-   왜 실패했는지 로그 기록
-   디버그 오버레이 표시 가능
-   플레이어에게 과도한 패널티를 주지 않는다.

------------------------------------------------------------------------

# 10. 접근성

Easy - 오차 확대

Normal - 기본 오차

Hard - 오차 축소

실력은 패턴과 보스가 시험하고, 입력 인식은 불필요하게 엄격하지 않다.

------------------------------------------------------------------------

# 12. 플레이 테스트

체크 항목

□ 오인식률 3% 이하

□ 입력 지연 체감 없음

□ 기기 변경 시 동일한 결과
