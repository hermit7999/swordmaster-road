# Difficulty Curve Bible

> 목적
>
> 이 문서는 게임의 난이도를 설계하는 문서이다. 목표는 '쉬운 게임'도
> '어려운 게임'도 아니다. **계속 도전하고 싶은 게임**을 만드는 것이다.

------------------------------------------------------------------------

# 1. 난이도 철학

난이도는 플레이어를 막기 위해 존재하지 않는다.

난이도는 플레이어의 성장을 보여주기 위해 존재한다.

플레이어는

"억울해서 졌다."

가 아니라

"다음에는 이길 수 있겠다."

라고 느껴야 한다.

------------------------------------------------------------------------

# 2. 난이도 공식

학습 ↓

성공

↓

익숙함

↓

새로운 도전

↓

실패

↓

학습

↓

성공

이 사이클을 반복한다.

------------------------------------------------------------------------

# 3. 스테이지별 난이도

## Stage 1 : 입문

목표 - 이동 익히기 - 기본 베기 - 첫 검술

성공률 목표 : 90%

------------------------------------------------------------------------

## Stage 2 : 이해

목표 - 회피 - 거리 조절

성공률 목표 : 80%

------------------------------------------------------------------------

## Stage 3 : 응용

목표 - 검술 연계 - 첫 오의

성공률 목표 : 70%

------------------------------------------------------------------------

## Stage 4 : 숙련

목표 - 빌드 활용 - 상황 판단

성공률 목표 : 60%

------------------------------------------------------------------------

## Stage 5 : 시험

목표 - 지금까지 배운 모든 것

첫 도전 성공률 목표 : 30\~40%

재도전 성공률 목표 : 70% 이상

------------------------------------------------------------------------

# 4. 실패 설계

실패는 학습 정보를 제공해야 한다.

플레이어는 패배 후

-   무엇이 문제였는지
-   무엇을 연습해야 하는지

알 수 있어야 한다.

------------------------------------------------------------------------

# 5. 보스 난이도

보스는 새로운 규칙을 만들지 않는다.

보스는 기존 규칙을 조합한다.

보스에게 죽은 이유는 항상 설명 가능해야 한다.

------------------------------------------------------------------------

# 6. 성장과 난이도

영구 성장은

실패를 줄여준다.

실력은

성공을 만든다.

둘이 함께 작동해야 한다.

------------------------------------------------------------------------

# 7. 금지 사항

-   원인 모를 즉사
-   운으로만 피하는 공격
-   체력만 많은 보스
-   패턴 없이 숫자만 높은 적
-   튜토리얼 없이 새 규칙

------------------------------------------------------------------------

# 8. 플레이 테스트 체크리스트

□ 첫 보스를 이해할 수 있는가?

□ 두 번째 도전에서 더 오래 버티는가?

□ 세 번째 도전에서 패턴을 읽는가?

□ 클리어 후 실력이 늘었다고 느끼는가?

□ 포기보다 재도전이 먼저 떠오르는가?
