# ADR-001

## 일반전과 보스전을 분리한다.

### 결정

필드는 RPG 스타일, 보스는 Duel Mode를 사용한다.

### 이유

-   일반전은 성장과 속도감을 제공한다.
-   보스전은 플레이어의 실력을 시험한다.
-   하나의 전투 방식으로 두 재미를 모두 만들기 어렵다.

### 변경 조건

두 전투 방식이 각각 충분히 재미있지 않을 때만 재검토한다.

------------------------------------------------------------------------

# ADR-002

## 한 판은 약 5분으로 설계한다.

### 결정

한 스테이지 = 약 5분

5개 스테이지 = 약 30\~40분

### 이유

-   반복 플레이가 빠르다.
-   로그라이크 구조와 잘 맞는다.
-   1인 개발자가 콘텐츠를 관리하기 쉽다.

------------------------------------------------------------------------

# ADR-003

## 실력이 중심이다.

### 결정

실력 : 성장 = 약 60 : 40

### 이유

-   컨트롤이 승패를 결정해야 한다.
-   하지만 성장 시스템이 초보 플레이어를 보조한다.

------------------------------------------------------------------------

# ADR-004

## 장비는 플레이 스타일을 바꾼다.

### 결정

공격력 +1000 같은 설계는 지양한다.

장비는

-   사거리
-   패링 효과
-   회피 효과
-   검술 특화

등 플레이 방식을 변화시킨다.

------------------------------------------------------------------------

# ADR-005

## 적은 선생님이다.

### 결정

모든 적은 플레이어에게 하나의 기술을 가르친다.

예시

-   방패병 → 패링
-   창병 → 거리
-   궁수 → 이동
-   검객 → Duel 감각

보스는 종합 시험이다.

------------------------------------------------------------------------

# ADR-006

## 개발 우선순위

1.  손맛
2.  일반전
3.  Duel Mode
4.  성장
5.  콘텐츠

콘텐츠를 늘리기 전에 손맛을 완성한다.

------------------------------------------------------------------------

# ADR-007

## 1인 개발 원칙

새로운 기능은 아래 질문을 통과해야 한다.

-   재미가 증가하는가?
-   구현 가능한가?
-   유지보수 가능한가?
-   출시를 늦추지 않는가?

YES가 충분하지 않으면 보류한다.
