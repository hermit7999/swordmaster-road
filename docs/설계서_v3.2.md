## 1. 개요
- 획(劃)을 긋는 제스처 검술 액션 RPG. 모바일+PC 웹, 가로 고정.
- 입력 2방식: **검로(劍路)** 궤적(정확도=궤적 정밀도) · **검결(劍訣)** 방향 커맨드(정확도=리듬).
- 판정=순수 함수(DOM 비의존, `src/engine/*.ts`). 수치=데이터 JSON(`src/data/*.json`). 이벤트 규격 `{strokeId, accuracy, grade, inputMode, timestamp}`.
- 등급 5단계: 미스/배드/굿/그레이트/퍼펙트.

## 2. 판정·입력 시스템

### 2.1 입력 채널 & 중재 (InputArbiter v2)
두 채널(검로=포인터, 검결=키보드/패드) **동시 활성**. **행동 점유(occupancy)** 모델:
- 먼저 시작한 채널이 행동 종료까지 점유(검로: pointerdown~endStroke, 검결: 첫 입력~resolveCommand).
- **v2(취소/우선순위, T1-07 확정)**: 아직 안 그린(점만 찍힌) 제스처는 **커맨드가 선점**(release+취소) → 빠른 결전 응수에서 의도 입력 드롭 제거. (구 v1은 무시(drop)만.)
- 백스톱 `arbiterStaleMs` 2000ms(영구 잠금 방지).
- 커맨드 확정: 완전 일치 즉시(리듬은 타임스탬프로 보존), 아니면 `commandResolveMs` 640ms 뒤. 동시 입력(카디널 쌍)은 `simulMs` 70ms 내면 대각으로 접음.

### 2.2 검로 파이프라인 `judgeStroke`
1. 좌수면 입력 x 미러(canonical=우수로 접기).
2. 탭/짧음 거부(rejectRatio 0.035 / **사전거부 바닥 = 템플릿 최소 minLenRatio**, 찌르기 0.05로 낮아짐).
3. 64pt 리샘플 → 크기 정규화(회전 비정규화).
4. **형태 매칭**(§3): 원(circle) vs 직선(line). 직선은 최근접 템플릿(dist>0.34 → 불명), 동률은 lenBand로 판별.
5. 정확도 = 방향 40% + 형태 30% + 속도 15% + 완주 15%.

### 2.3 점수 산식
| 성분 | 가중 | 산식 |
|---|---|---|
| 방향 | 40% | 구간별 국소 방향과 템플릿 방향 내적 clamp(0..) 평균. **circle=회전방향 일치+단조성, 반대방향=0** |
| 형태 | 30% | line=(정규화 현/경로)². **circle=원형도(§3.2)** |
| 속도 | 15% | 이상 구간(획별 speedMs 오버라이드 가능, 기본 120~700ms) 밖 선형 감쇠 |
| 완주 | 15% | line=정규화 끝점 정합. circle=스윕/360 |

### 2.4 검결 리듬 `judgeRhythm`
- 이상 비트 대비 타이밍(퍼펙트±50/그레이트±100/굿±180/배드±280ms).
- **동시 그룹(동일 이상 오프셋 연속) = 순서 무관 멀티셋 매칭** — chord ↓+→에서 →가 먼저 들어와도 정상(T1-03 버그 수정). 순차 그룹은 시간순.
- self-referenced(첫 입력=다운비트). 위력 +10% 조건: raw≥2 && 등급≠미스 && 최대오차≤100ms(패드 단일 대각 제외).

### 2.5 유파 (좌표 기반 미러 — 확정)
`STYLES`: `mirrorX` + `updownCluster`. **미러는 좌표 기반**(입력 x 미러로 canonical 공간에 접기) — 판정 결과 strokeId는 항상 canonical. **v2 §2의 "획ID 매핑 테이블 단순화" 제안은 철회**: 원(wonmu) 손방향(chirality)에서 ID맵이 어긋나므로 좌표 미러가 더 정확. 결과적으로 **검술 조합을 canonical로 정의하면 두 유파 자동 동작**(연풍참 좌수판 오버라이드 불필요). `StyleManager`(src/engine/style.ts): 미러 획ID맵(표시용)·techniqueCombo(미래 비미러 유파용 오버라이드 지점)·updownCluster·otherStyle.

## 3. 형태 판정 (M1)
### 3.1 shapeType
획 데이터에 `shapeType`("line"|"circle"). 원 게이트 = 스윕≥`circleMinSweepDeg`(200°) && 반지름CV≤`circleMaxCV`(0.35) && 폐합≤`circleMaxClosure`(0.6) — 호/스파이럴은 원 아님.
### 3.2 원형도(circle 형태 점수)
반지름 일관성(CV, `radiusCVZero`=0.5로 0점) 50% + 각도 스윕(`sweepFullDeg`=330° 만점) 30% + 폐합(gap/둘레, `closureZero`=0.5로 0점) 20%.
### 3.3 lenBand 판별
형태 동률(dist ≤ best+`classifyTieEps` 0.02) 후보를 캔버스 대각 대비 길이비 밴드로 구분. 찌르기 [0.035,0.16] vs 횡베기 [0.16,1.0]. 트레이드오프: 경계 근처 짧은 횡베기가 찌르기로 갈 수 있음(실측 0.5%).
### 3.4 획별 오버라이드
`minLenRatio`/`speedMs`/`lenBand`를 strokes.json에서 획별 오버라이드(미지정 시 balance 전역). 코드 하드코딩 없음.

## 4. 검술/전투 수치 (M1)
### 4.1 검술 6종 (techniques.json)
| 검술 | 조합 | cost | base데미지 | 비고 |
|---|---|---|---|---|
| 일자베기 | 횡(좌→우)→횡(우→좌) | 10 | 18 | |
| 십자베기 | 횡(좌→우)→내려찍기 | 12 | 22 | 경직 |
| 연풍참 | 횡(좌→우)→사선(↘) | 15 | 28 | 광역 |
| 낙뢰참 | 올려베기→내려찍기→찌르기 | 22 | 45 | 단일 고위력 |
| 회천무 | 원무→횡(좌→우)→횡(우→좌) | 25 | 40 | 전방위 |
| 섬광자돌 | 사선(↘)→사선(↙)→찌르기→찌르기 | 30 | 60 | 관통 |

데미지 ≈ 코스트×1.8~2.0(아츠가 기본 획보다 행동당 효율 우위). 기본 획(마나 무소모) 데미지 = `basicAttackDamage` 10.
### 4.2 곱셈 체인 & 마나
데미지 = base × (평균≥95?1.2:1.0) × 검결보너스. 검결 +10% = **아츠 단위 all-or-nothing**(`commandBonusMode`, proportional 전환 가능). 마나 = cost×(2−평균/100).
### 4.3 미스 페널티
진행 중 미스 = 발동 실패, 마나 cost×`missPenalty.manaFraction`(0.3) 소실 + 경직 `missPenalty.stunMs`(800ms). **페널티 대상 = 진행 중 접두어 후보 중 최장**(공유 접두어 모호성 규칙 — 문서화).
### 4.4 다중 검술 트래커
접두어 공유 처리(긴 조합 우선: 회천무>일자베기), 기본 획 노이즈 prune, HUD에 진행 후보 표시.

## 5. 씬 아키텍처 = DOM + Canvas (Phaser 보류)
- 씬(수련/결전/승급/자유연습)은 DOM 오버레이 + 단일 Canvas(잉크 궤적/가이드/오버레이)로 구현. `emitStroke`가 모드별 라우팅(combat→trial→training→자유).
- **Phaser 이관 보류**(NFR-011): ① 엔진 순수 TS 분리로 이관 비용 통제 ② 잉크 라인 판타지(정지 일러스트+이펙트)는 현 스택 충분 ③ M2 일정 리스크. **명확한 성능/연출 한계 도달 시 멈추고 보고 후 재평가.**
- 결전 오디오 큐: 공격 방향별 톤(→520/←392/↑720/↓300Hz) — 소리만으로 응수 방향 인지.

## 6. 데이터 파일 (src/data)
- `strokes.json`: 획 **10종**(shapeType/path/command/raw/rhythm/lenBand + minLenRatio/speedMs/**sectorHalfDeg** 오버라이드). 직선 8방향(↗↖ 포함)+찌르기+원무. 직선 분류=net 각도 섹터(±22.5°, 획별 오버라이드), 방향점수=국소 일치도×이상각 오차.
- `techniques.json`: 검술 6종(combo/window/cost/damage + stun/aoe/pierce).
- `styles.json`: 유파(mirrorX/updownCluster + 선택 techniqueOverrides).
- `enemies.json`: 적(hp/attacks[name/dir/counter/counterName/damage]).
- `trials.json`: 승급 시험(strokes/intervalMs/avgPass).
- `balance.json` 신규 필드: `basicAttackDamage`·`commandBonusMode`·`missPenalty`·`classifyTieEps`·`shape`(circleMinSweepDeg/circleMaxCV/circleMaxClosure/radiusCVZero/closureZero/sweepFullDeg/circleWeights)·`combat`(playerHp/manaMax/startMana/parryDamage/manaRecover*/observeMs/respondMs)·`simulMs`·`commandWindow`·`commandResolveMs`·`arbiterStaleMs`·`rhythm`(beatMs/windows).
