### (d) M1 기술 부채 / 리스크
1. **InputArbiter = '무시(drop)' 모델** — 겹친 후발 입력을 큐잉·취소 처리 안 함. 빠른 결전 페이즈에서 의도 입력이 드롭될 수 있음 → M1에서 우선순위/큐/캔슬 정식화.
2. **검결 리듬은 self-referenced**(첫 입력=다운비트) — 외부 메트로놈 클럭에 sync 안 됨. 결전 '합' 페이즈에서 글로벌 비트 동기가 필요하면 재설계.
3. **판정 순수부 + DOM 계층이 단일 index.html에 혼재** — M1 Phaser3+TS 이식 시 engine(순수부) 모듈 분리 + BALANCE/STROKE_TEMPLATES/TECHNIQUES/STYLES를 JSON 외부화. (현재 데이터블록은 JSON화 가능한 형태로 유지 중.)
4. **좌수 미러 시 새 획이 되는 케이스 미구현** — 사선 ↘→↙(diag_dl)는 M0에 없음. M1에서 좌수검류 전용 획 세트 정의 필요.
5. **제스처 템플릿=3획 직선뿐** — 원무(원)·찌르기(짧은 직선)는 분류 임계/속도 재튜닝 필요. 특히 찌르기는 minLenRatio(0.12)와 충돌 소지(짧은 획이 '거부'될 위험).
6. **powerBonus 실데미지 미연결** — 현재 표기만. M1 전투 위력 계산에 실제 곱셈 반영 필요.
7. **콤보창(1500) vs commandResolveMs(640) 상호작용** — 검결 2입력이 640ms까지 늘어져 콤보창을 잠식. 빠른 콤보 요구 시 튜닝.
8. **AudioContext 사용자 제스처 게이팅** — 첫 입력 전 메트로놈 켜면 브라우저 정책상 소리 지연 가능.
9. **테스터 플레이테스트 미실시** — DoD "다시 긋고 싶다" 및 "납득 가능한 실패" 체감은 실사용자 검증 남음(리스크 1순위 대응 항목).
